Vue模式及相关指令

发表于 2019-04-09

MVVM模式
1.Vue将每一个页面分割成 M ，V， MV。其中MV是核心。MV是M层与V层的调度者。
每当用户进行操作如果需要业务处理，都会经过网络请求，去请求后端的服务器，此时这个请求就会由后面的App.js监听到。
M层
M层是保存的每个页面中单独的数据

VM层
它是一个调度者，分割了M与V，每当V想要获取M中的保存的数据时，都要进过VM层处理

V层
视图层是每个页面中的Html结构

M-MV-V是取数据，从M到V，对Html进行渲染
V-MV-M是存数据，存数据

主要是MVVM实现了数据的双向保定 其中VM是核心。可以让开发更方便。

2.Vue的使用
在Html中引入Vue的包
再导入包后，在浏览器的内存中就有了Vue的构造函数

})

选择的id = app是MVVM中的V层，view实例控制的元素区就是V。
new 出来的Vm对象是MVVM中的VM调度者。 vm实例
data是页面中的data中的model，用来保存每个页面的数据
3 v-bind绑定的意思。
v-bind是绑定属性的指令例如 绑定按钮的title属性 v-bind：title，其中v-bind
v-bind:可以简写为:要绑定的属性。
v-bind可以写合法的JS表达式

v-on指令是用来绑定事件的 简写为@

v-on：click
click需要在data中定义。
methods：{}属性中定义了当前Vue实例的方法。
事件修饰符，stop阻止冒泡，prevent阻止默认事件，capture捕获， self只在该元素本身触发事件，once只触发一次事件

箭头函数

发表于 2019-03-14

1.箭头函数
相当于匿名函数，对函数声明简化。形式是 x=> {},x是参数，后面中的大括号是函数体，如果参数不是一个，则需要大括号。
（1 能够让函数声明更加简短
（2 不绑定this
2.Promise
用于异步任务，可以作为对象也可以作为方法。有三个状态
1.pending初始状态，既不是成功也不是失败
2.fulfilled操作成功
3.rejected意味着操作失败。
new Promise(function(resolve, reject))
调用resolve与reject函数被调用则状态立马改变。

一般是想让一个函数有Promise功能则让函数返回一个Promise对象

活动页布局

发表于 2019-03-11

1.活动页面的原理
主要使用overflow：hidden隐藏超出的页面，给整个页面在一个div标签中，div赋值设置top（向下翻页）或者left（向右翻页），每一页都是固定的距离。这样可以在一个html文件中完成多个页面的展示。
2.图片库原理
使用div标签建立一个大图展示区域，分别使用div建立小图展示区域。给小图定位赋值使用float：left属性，向左浮动，使他们排成一列。简单的页面布置就这样成功了，再加入图片的时候，使用createElement（’img’）方法建立img标签，并建立for循环，使用setattribute（’src’,路径）由于路径是字符串，所以要使用拼接字符串的方式，提前将图片的名称设置为1-5.jpg。

实现点击小图更新相应的大图
给小图区域的父元素绑定一个onclick事件，本来是要给每一个img绑定事件，这里使用了事件冒泡，内部子元素要绑定相同的事件，则可以绑定在他们的父元素上，事件也可以触发，减少代码量。使用getattribute（’src’）方法获取当前点击图片的src路径。，并使用setattribute赋值给大图的src路径，则可以实现点击小图切换到对应大图上。
