盒模型

发表于 2019-03-02

1.盒模型
（1）在CSS样式中每一个元素在处在一个盒模型中，width和height设置内容区，里面包含了你想呈现的内容，一段文字或者一个图片等等。
（2）这个盒子还有3个属性：padding border margin

padding：内边距，指的是元素内容区和边框以内的空间大小，单位为px，顺序为上右下左。例如：padding：10px 20px 30px 40px.用于设定元素与你的内容之间的距离，注意不能使用负值，也可以使用百分比赋值。

border：边框，在元素周围创建边框，是一个边缘。使用，border-width，border-height等等设置。

margin：外边距，是设置元素与周围元素的距离属性，可以使用百分比赋值，是相对于父元素width或height的距离百分比，和padding一样，参数是从上右下左赋值

对于IE的盒子模型，与一般都不同，普通的盒子宽度=width+padding+border+margin，IE盒子模型宽度=width+margin

特殊情况：外边距合并：当两个垂直外边距相遇则合并成一个外边距，合成后的取两者中的大值。行内框，浮动框，绝对定位不会发生外边距合并。
2.定位 绝对定位和相对定位
块级元素：每个元素都是从新的一行开始，可以容纳内联元素和其他块级元素，盒模型属性都可以控制。
行内元素：和其他元素在一行上，盒子模型距离不能改变，只能容纳文本，或者其他内联元素
可以设置display：block属性改变元素的属性
普通文档流将浏览器窗体自上而下分成一行行，在每行按照从左到右排放元素。
position属性：static，relative，absoulte，fixed

static：元素框正常生成，块级元素生成矩形框，成为文档流的一部分，行内元素生成行框嵌套在父容器中

relative：元素框偏移某个距离，元素仍保持为定位前的形状，原本的占位空间保留。元素模型不能重叠，会导致它覆盖其它元素

absolute：元素框从文档流中删除，并相对于其包含块定位，原来的所占空间会关闭，元素定位后生成一个块级框，不论在原来文档流中是何类型的框。元素模型可以重叠，设置z-idnex参数。设置移动参数会影响其他元素的位置

fixed：类似于absolute，不过其包含块是视窗本身也就是窗口。

float：浮动，使元素脱离文档流向左或者向右移动，可以在一行中设置两个div标签。通常一个div标签占一行，overflow：auth或者none可以清除浮动

(4) position的用法
1.relative（相对的）它允许指定CSS应如何相对于其再也米娜正常流中的当前位置移动它，通常与left，right，bottom设置搭配。
2.absolute（绝对的）它将元素锁定在父容器的位置，与（相对的）位置不同，从文档的正常流中删除元素，周围的项忽略它，CSS的偏移属性（顶部底部左右）用来调整位置
3.fixed（绝对定位）用于锁定于相对于浏览器窗口的元素。与absolute相似，但关键区别是当用户滚动时，具有固定位置的元素不会移动。一般使用于导航栏。
4.float（浮动定位）它并不是使用position设置，浮动元素将从文档的正常流中删除，并推送到父元素的left或right，它通常用于width属性，指定浮动元素所需水平空间。
（5）z-index用法，用于设定元素层级，层级高的元素会展示在层级低的元素之上。如z-index:2
可以同过设置margin：auto 可以使元素居中。
（6）background：linear-gradinet（）颜色渐变设置。background:linear-gradient(90deg, red, yellow, rgb(204, 204, 255))。
（7）transform属性：scale函数:缩放,skew函数单位deg：倾斜,rotate函数单位deg：旋转,translate函数：移动单位px,transform-origin基准点函数,单位px

'冒泡事件和捕获事件'

发表于 2019-01-12

事件冒泡：指的是当事件触发后，从被绑定的元素内部逐级向外传播，用于事件代理，事件委托。在子元素上触发的事件，绑定到父元素上，经过事件冒泡
处理同意类元素具有相同事件的。从内向外冒泡，绑在子元素的事件与绑在父元素的事件效果相同。适合委托的事件：click， mousedown，mouseUp，等等事件
事件捕获：和冒泡相反，是从外及内的
例如：
‘’’






‘’’
js事件
var id1 = document.querySelector(‘#id1’)
var id2 = document.querySelector(‘#id2’)
var id3 = document.querySelector(‘#id3’)
//事件冒泡
id1.addEventListener(‘click’, function(event){
console.log(‘click id1’, event)
})
id2.addEventListener(‘click’, function(event){
console.log(‘click id2’, event)
})
id3.addEventListener(‘click’, function(event){
console.log(‘click id3’, event)
})
//则它触发顺序为id3，id2，id1
//事件捕获
id1.addEventListener(‘click’, function(event){

}, true)

console.log(‘click id2’, event)
}, true)
id3.addEventListener(‘click’, function(event){
console.log(‘click id3’, event)
, true})

JS中每一个变量（也就是每一个值）都是一种数据类型
（一）内置的基本数据类型有以下几类：
1.number 数字 2.string 字符串
3.boolean 布尔变量 4.object 对象
（二）1.8
笔记–类型-值-变量
1.数据类型：在编程语言中表示能够操作的值被称为数据类型
2.编程最基本的特性就是能够支持多种数据类型
3.通过将值赋值给变量将值保存在变量中
4.变量是一个名称，可以通过名称来获得对值的引用
数据类型分类
1.JavaScript的数据类型分为两类：原始类型（primitive type）和对象数据类型（object type）
原始数据类型包括：数字，字符串，布尔值。特殊的原始值：null和undefined他们不是原始数据类型
对象数据类型：除了原始数据类型，null和undefined之外都是对象数据类型。（1）对象是属性（property）的集合，每个属性由名和值组成，例如{小明：boy}普通的对象是命名值的无序集合。（2）数组也是对象数据类型，它是带编号的有序集合。（3）特殊对象：函数-函数是具有与它相关的可执行的代码的对象，通过调用函数来运行可执行代码。
2.JavaScript解释器有自己的内存管理工具，可以自动进行垃圾回收
3.JavaScript是一种面向对象的语言，在JavaScript中只有null和undefined是无法拥有方法的值。
4.JavaScript的类型除可以分为3种（1）原始类型和对象类型。（2）拥有方法类型和不拥有方法类型。（3）可变类型和不可变类型，可变类型：比如对象和数组的值是可以修改的，不可变类型：比如null，字符串，undefined是不可修改的
5.JavaScript的变量：变量是无类型的，变量可以被赋值任何值，也可以重新赋予不同类型的值例。不在任何函数内声明的变量为全局变量，在程序任何地方都是可见的。
