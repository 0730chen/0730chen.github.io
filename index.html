使用hugo部署网站步骤

  1. 下载安装hugo
    • 通过链接下载hugo安装包
    • 解压安装hugo后生成hugo.exe文件
    • 将hugo.exe路径添加到系统的path路径中
    • 在命令行中输入hugo version 查看是否安装成功
  2. 生成静态站点
    • 输入命令 hugo new site blog 建立文件夹名称
    • hugo new blog.md 命令生成页面
    • hugo new posts\first-post.md 添加页面
    • 启动 hugo server 启动服务
  3. 在Github中新建一个your-github-name.github.io的仓库
    • 将根目录下config.toml中文件baseURL改变成网址
    • 使用git将本地仓库与github云端仓库连接起来
    • git init 初始化
    • git remote add origin 链接
    • git add .
    • git commit -v
    • git push -u origin master
      文件都储存在public文件中
  4. 购买域名 购买soilname或者阿里云购买域名
    • 点击github.io仓库中的setting，填入你购买的域名
    • 在你的github.io仓库中出现一个CNAME文件
    • 解析域名，将域名指向你的github.io
  5. 访问购买的域名，出现刚才搭建的博客

MVVM模式
1.Vue将每一个页面分割成 M ，V， MV。其中MV是核心。MV是M层与V层的调度者。
每当用户进行操作如果需要业务处理，都会经过网络请求，去请求后端的服务器，此时这个请求就会由后面的App.js监听到。
M层
M层是保存的每个页面中单独的数据

VM层
它是一个调度者，分割了M与V，每当V想要获取M中的保存的数据时，都要进过VM层处理

V层
视图层是每个页面中的Html结构

M-MV-V是取数据，从M到V，对Html进行渲染
V-MV-M是存数据，存数据

主要是MVVM实现了数据的双向保定 其中VM是核心。可以让开发更方便。

2.Vue的使用
在Html中引入Vue的包
再导入包后，在浏览器的内存中就有了Vue的构造函数

msg：’hello’
} data属性是从存放的el中要用到的数据。,以前要使用DOM操作区添加数据，现在是使用vue提供的指令就可以将数据渲染到页面上，Vue不提倡去操作DOM元素。使用引用data中的数据
})

选择的id = app是MVVM中的V层，view实例控制的元素区就是V。
new 出来的Vm对象是MVVM中的VM调度者。 vm实例
data是页面中的data中的model，用来保存每个页面的数据

v-bind是绑定属性的指令例如 绑定按钮的title属性 v-bind：title，其中v-bind
v-bind:可以简写为:要绑定的属性。
v-bind可以写合法的JS表达式

v-on指令是用来绑定事件的 简写为@

v-on：click
click需要在data中定义。
methods：{}属性中定义了当前Vue实例的方法。
事件修饰符，stop阻止冒泡，prevent阻止默认事件，capture捕获， self只在该元素本身触发事件，once只触发一次事件

1.箭头函数
相当于匿名函数，对函数声明简化。形式是 x=> {},x是参数，后面中的大括号是函数体，如果参数不是一个，则需要大括号。

2.Promise
用于异步任务，可以作为对象也可以作为方法。有三个状态
1.pending初始状态，既不是成功也不是失败
2.fulfilled操作成功
3.rejected意味着操作失败。
new Promise(function(resolve, reject))
调用resolve与reject函数被调用则状态立马改变。

一般是想让一个函数有Promise功能则让函数返回一个Promise对象
